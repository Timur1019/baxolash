# Роли и права доступа

При любой доработке, связанной с ролями (клиент / сотрудник компании), учитывать правила ниже.

## 4.1 Клиент (банк / сотрудник банка)

**Кто:** сотрудники банка, организации, заказывающие оценку.

**Может:**
- создавать заявку на оценку;
- загружать документы;
- смотреть статус;
- скачивать отчёт об оценке;
- видеть стоимость оценки;
- общаться через файлы (загрузка/скачивание документов).

**Не может:**
- менять статус после передачи заявки;
- редактировать чужие заявки;
- назначать оценщиков.

**Фронт:** раздел `/client` (ClientLayout). Роуты: `client-requests`, `client-request-new`, `client-request-detail`.  
**Бэкенд:** роль `CLIENT_EMPLOYEE`; доступ только к своим заявкам (`clientUser.id === currentUser.id`).

---

## 4.2 Сотрудник оценочной компании

**Кто:** оценщики, специалисты оценочной компании.

**Может:**
- видеть все поступившие заявки;
- фильтровать заявки;
- открывать карточку заявки;
- редактировать данные заявки;
- менять статус;
- загружать отчёт;
- указывать стоимость оценки;
- подтверждать завершение.

**Фронт:** раздел `/company` (CompanyLayout). Роуты: `company-requests`, `company-request-detail`.  
**Бэкенд:** роль `COMPANY_EMPLOYEE` (и `ADMIN` для доступа в панель компании); доступ ко всем заявкам, право менять статус/стоимость/отчёт и подтверждать завершение.

---

## Редирект после входа

- `ADMIN` → `/admin`
- `CLIENT_EMPLOYEE` → `/client/requests`
- `COMPANY_EMPLOYEE` → `/company/requests`

Роутер проверяет `meta.role`: доступ в `/admin` только для ADMIN, в `/client` только для CLIENT_EMPLOYEE, в `/company` для COMPANY_EMPLOYEE и ADMIN.
